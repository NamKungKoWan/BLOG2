撰寫BRD/MRD/PRD文檔

BRD-這麽做的好處，並説明好處在哪裏。

MRD-通過BRD明確了這個事情值得一做後，描述應該這麽做，並説明這麽做的原因。

PRD-獲得了授權，而且已經確定了要走的路綫，剩下的就是打造產品了

  • 版本號的命名規則

    • 歸零原則：前一個數字增加一位，後面的數字都歸零
    • 收費原則：子版本號和修正版本號的變化，一般看做版本内升級，附加收費用，版本號變化則加收費用

  • 歷史修訂

    • 編號
    • 版本號
    • 修訂章節
    • 修訂原因
    • 修訂日期

        • 修訂人
